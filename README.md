## 🤖 How to use it

This repository uses a two-step workflow.

### 1. Load the senior developer persona

Open `SOP-Senior-Developer-Persona.md`.

Paste the full file into your AI assistant first. This gives the assistant a senior technical architect role with a focus on:

- Security
- Code quality
- Measurable findings
- Clear report structure
- Effort estimates
- Risk scoring

Use it as the first message or system prompt if your AI tool allows that.

### 2. Run the audit process

Open `SOP-AI-Assisted-Code-Audit.md`.

Follow the checklist in order:

1. Gather the source code
2. Gather dependency files
3. Gather build settings
4. Gather any past audit reports
5. Run the pre-analysis commands
6. Share the results with the AI assistant
7. Review the findings and next steps

## 🔍 What to prepare before an audit

Before you start, gather these items:

- The full codebase
- `package.json`, `requirements.txt`, or similar dependency files
- Build files such as `Dockerfile`, `Makefile`, or CI config
- Test files
- Deployment settings
- Any old audit notes

This helps the AI see the full project instead of a small slice of it.

## 📌 Good habits during the audit

- Keep the code sample large enough to show real context
- Include build and dependency files
- Use the same workflow each time
- Save the report in a shared folder
- Track findings with IDs
- Note the effort needed for each fix

## 🧾 Expected output from the AI review

A strong audit report should include:

- Finding ID
- Issue name
- Risk level
- Affected files
- Why it matters
- Fix effort
- Suggested next step

This helps you turn a long code review into a clear action list.
